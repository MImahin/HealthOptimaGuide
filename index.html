HealthOptima

‚ñº What is it?

HealthOptima is an intelligent health analysis platform designed to empower users with personalized food and wellness insights based on their medical conditions and lifestyle data.

It enables users to upload food ingredient images, from which nutritional data is extracted using advanced Optical Character Recognition (OCR). This data is then cross-checked against medical conditions like diabetes, hypertension, allergies, and more, offering condition-specific dietary feedback in real-time.

Beyond dietary suggestions, HealthOptima provides a comprehensive health analytics dashboard. It calculates and visualizes key body metrics ‚Äî including BMI, BMR, TDEE, and body fat percentage ‚Äî alongside recommended calorie intake tailored to the user‚Äôs profile.

Users can also instantly search any food by name, receiving detailed nutrient breakdowns and interactive pie charts. This supports data-driven meal planning, clearly showing what to consume, moderate, or avoid ‚Äî along with ingredient warnings for sensitive conditions.

Comparative analytics are available to benchmark one's health status against the broader user base, including age-, gender-, and weight-specific visualizations showing how common or rare their health conditions are across populations.

Finally, HealthOptima integrates health expert guidance and curated video content to kickstart and sustain your personal wellness journey ‚Äî making it not just a tool, but a smart partner for long-term health.

‚ñº How to set up?

Clone the GitHub repository:

git clone https://github.com/MImahin/HealthOptima.git

Navigate to the project directory:

cd HealthOptima

Get your own API key:

This project uses the USDA FoodData Central API. To use it, you need to obtain a personal API key from the official site.

  • Visit https://fdc.nal.usda.gov/api-guide
  • Go to "Gaining Access"
  • Sign up with your email and follow the instructions
  • Once you receive your key, proceed to the next step

After obtaining your API key, create a .env file in the root of the project and add:

API_KEY=your_usda_api_key_here

Note: This file is used locally and is excluded from version control (Git) for security.

Create and activate a virtual environment (Windows):

python -m venv venv
venv\Scripts\activate

Or on macOS/Linux:

python3 -m venv venv
source venv/bin/activate

Install required dependencies:

pip install -r requirements.txt

Run the app locally:

python app.py

Access the app in your browser:

http://localhost:5000

Troubleshooting Tips & Common Errors

Error: Dependency version conflicts or installation failures

What it means:
Sometimes the versions of packages in requirements.txt might not be compatible with your system or other installed packages. This can cause installation errors or runtime issues.

How to fix it:

  • Open the requirements.txt file in a text editor.
  • Find the packages causing issues (check your error logs).
  • Remove the version numbers after the package names (for example, change package==1.2.3 to just package).
  • Save the file.
  • Run the command again:
pip install -r requirements.txt

This lets pip install the latest compatible version.

Error: 'python' is not recognized as an internal or external command

What it means:
Your system doesn‚Äôt recognize the python command because Python‚Äôs executable isn‚Äôt in your system PATH or you‚Äôre on a system where the command is python3.

How to fix it:

  • Try running your commands using python3 instead of python.
  • Make sure Python is installed on your machine.
  • Add Python to your system PATH:
    • On Windows, add Python‚Äôs install directory to the Environment Variables.
    • On macOS/Linux, Python is usually pre-installed or installed via package managers.
  • You can verify Python installation by running:
python --version
python3 --version

Error: Virtual environment activation command not working

What it means:
Your shell/terminal is unable to run the activation script for the Python virtual environment.

How to fix it:
Check that you are using the correct command for your operating system:

  • Windows (Command Prompt):
venv\Scripts\activate
  • Windows (PowerShell):
    Note: PowerShell may block script execution by default; you might need to run PowerShell as administrator and run:
.\venv\Scripts\Activate.ps1
Set-ExecutionPolicy RemoteSigned
  • macOS/Linux:
source venv/bin/activate

Error: Port 5000 already in use

What it means:
Another program or previous instance of your app is already running and using port 5000, which is Flask‚Äôs default port.

How to fix it:

  • Close the program or process using port 5000. On Windows, you can use Task Manager to end Python or related processes. On macOS/Linux, use commands like:
lsof -i :5000
kill <PID>
  • Alternatively, run your Flask app on a different port by specifying the port number in your run command:
python app.py --port=5001

Or modify your Flask app to specify a different port inside the code.
